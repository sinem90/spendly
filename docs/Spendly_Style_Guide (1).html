Spendly

Design System & Style Guide

Purpose: This style guide defines the visual design language for Spendly, a personal finance analytics platform. Consistent application of these styles ensures a cohesive, professional, and user-friendly interface that builds trust with users managing their financial data.

Design Philosophy: Spendly's design prioritizes clarity, accessibility, and confidence. The color palette conveys professionalism and trustworthiness appropriate for financial applications, while maintaining enough visual interest to keep the interface engaging. Typography emphasizes readability, and UI components are designed for intuitive interaction.

Color Usage Guidelines: Use semantic colors intentionally to communicate meaning. Buttons

Buttons are the primary method of user interaction. Each style communicates different levels of importance and action types.

Button Variants

Button Usage:
• Primary: Main actions (Save, Submit, Add Transaction)
• Secondary: Alternative actions (Cancel, Back)
• Success: Positive confirmations (Confirm, Complete)
• Danger: Destructive actions (Delete, Remove)
• Outline: Secondary emphasis without background
• Ghost: Tertiary actions, navigation

Button States

Hover State: Darker shade + subtle upward translation + shadow
Active State: Pressed appearance with reduced shadow
Disabled State: Reduced opacity (0.5), no hover effects, cursor: not-allowed
Focus State: Blue outline for keyboard navigation accessibility

Form Elements

Form elements are designed for clarity and ease of data entry, critical for financial transaction logging.

Form Guidelines: Labels should be clear and positioned above inputs. Use placeholder text sparingly for examples, not instructions. Provide helpful error messages inline. Focus states should be clearly visible with the primary blue color and subtle shadow.

Icons & Visual Elements

Icons should be simple, recognizable, and consistent in style. Use 24px as the standard size.

$ ✓ ! ✕
Icon Guidelines: Icons should match the color system. Use colored backgrounds (light variants) with darker icon colors for emphasis. Maintain consistent sizing (24px standard, 16px for small, 32px for large). Consider using icon libraries like Heroicons or Feather for consistency.

Accessibility Considerations

Color Contrast: All text must meet WCAG AA standards (4.5:1 for normal text, 3:1 for large text)
Focus States: All interactive elements must have visible focus indicators for keyboard navigation
Touch Targets: Minimum 44x44px for touch targets on mobile devices
Semantic HTML: Use proper heading hierarchy and semantic elements
Screen Readers: Include appropriate ARIA labels for complex interactions

Responsive Breakpoints

Mobile: 0 - 640px
Tablet: 641px - 1024px
Desktop: 1025px and above

Approach: Mobile-first design with progressive enhancement. Core functionality must work on all screen sizes. Consider touch-friendly targets on mobile (minimum 44x44px).
